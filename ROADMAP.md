# Roadmap

The roadmap is intentionally modest. The project should remain a self-hosted
personal blog system rather than grow into a general-purpose publishing
platform.

## Near Term

- Improve admin comment review ergonomics.
- Add more backend tests for categories, collections, sidebar, and social links.
- Add production backup/restore smoke-test notes after real deployment use.
- Add screenshots under `docs/assets/`.

## Later

- Optional Docker image publishing workflow.
- Optional email notification for approved/rejected comments.
- Draft and publish workflow improvements for posts.
- More editor usability improvements without changing the minimal visual style.
- Optional OpenAPI-generated frontend type checks if it stays lightweight.

## Not Planned by Default

- Hosted SaaS mode.
- External anti-spam API dependency.
- Automatic production database migrations on container start.
- Large UI framework migration.
- Public default administrator account.
